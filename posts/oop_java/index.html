Java OOP

Java là một ngôn ngữ lập trình hướng đối tượng (OOP - Object-Oriented Programming), và nó hỗ trợ các nguyên tắc cơ bản của OOP. Dưới đây là một cái nhìn tổng quan về các khái niệm OOP trong Java:

Các đặc điểm nổi bật của Java:

  1. Hướng đối tượng: Java hỗ trợ các nguyên tắc lập trình hướng đối tượng như kế thừa, đa hình, đóng gói và trừu tượng, giúp tổ chức mã nguồn một cách hiệu quả và dễ bảo trì.

a.Lớp (Class)

Lớp là một khuôn mẫu (template) để tạo ra các đối tượng. Nó định nghĩa các thuộc tính và phương thức mà các đối tượng của lớp đó sẽ có. Ví dụ:

public class Dog {
    String name;
    int age;

    void bark() {
        System.out.println("Woof!");
    }
}

b.Đối tượng (Object)

Đối tượng là một thể hiện (instance) của lớp. Mỗi đối tượng có trạng thái và hành vi riêng. Ví dụ:

Dog myDog = new Dog();
myDog.name = "Buddy";
myDog.age = 3;
myDog.bark(); // In ra "Woof!"

c.Đóng gói (Encapsulation)

Đóng gói là một nguyên tắc của lập trình hướng đối tượng nhằm che giấu thông tin chi tiết của đối tượng và chỉ cho phép truy cập thông qua các phương thức công khai. Điều này giúp bảo vệ dữ liệu và giảm sự phụ thuộc giữa các thành phần của chương trình.

d.Kế thừa (Inheritance)

Kế thừa là một trong những nguyên tắc cơ bản của lập trình hướng đối tượng (OOP). Nó cho phép một lớp (lớp con) kế thừa các thuộc tính và phương thức từ một lớp khác (lớp cha). Điều này giúp tái sử dụng mã và tạo ra cấu trúc phân cấp giữa các lớp.

Đặc điểm của Kế thừa

  • Tái sử dụng mã: Kế thừa cho phép lớp con sử dụng lại mã của lớp cha, giảm thiểu sự trùng lặp.
  • Tính mở rộng: Lớp con có thể mở rộng hoặc thay đổi hành vi của lớp cha bằng cách ghi đè các phương thức.

Ví dụ về Kế thừa

public class Animal {
    void eat() {
        System.out.println("This animal eats food.");
    }
}

public class Dog extends Animal {
    void bark() {
        System.out.println("Woof!");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog myDog = new Dog();
        myDog.eat(); // Phương thức từ lớp cha
        myDog.bark(); // Phương thức riêng của lớp con
    }
}

Đa hình (Polymorphism)

Đa hình là một trong những nguyên tắc cơ bản của lập trình hướng đối tượng (OOP). Nó cho phép các đối tượng khác nhau có thể được xử lý thông qua cùng một giao diện. Đa hình giúp tăng tính linh hoạt và khả năng mở rộng của mã nguồn.

Các loại Đa hình

  1. Đa hình thông qua phương thức ghi đè (Method Overriding):

    • Đây là khi một lớp con cung cấp một triển khai cụ thể cho một phương thức đã được định nghĩa trong lớp cha. Khi gọi phương thức từ một đối tượng của lớp con, phương thức ghi đè sẽ được thực thi.
    class Animal {
    void makeSound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("Woof!");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal myAnimal = new Dog();
        myAnimal.makeSound(); // In ra "Woof!"
    }
}

  2. Đa hình thông qua phương thức nạp chồng (Method Overloading)

Đây là khi một lớp có nhiều phương thức cùng tên nhưng khác nhau về tham số (số lượng hoặc kiểu dữ liệu của tham số). Khi gọi phương thức, trình biên dịch sẽ xác định phương thức nào sẽ được thực thi dựa trên các tham số được truyền vào.

class MathUtils {
    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }
}

public class Main {
    public static void main(String[] args) {
        MathUtils math = new MathUtils();
        System.out.println(math.add(5, 10)); // In ra 15
        System.out.println(math.add(5.5, 10.5)); // In ra 16.0
    }
}

Kết Luận

Đa hình là một tính năng mạnh mẽ của lập trình hướng đối tượng, giúp tăng tính linh hoạt và khả năng mở rộng của mã nguồn. Bằng cách sử dụng đa hình, bạn có thể viết mã dễ bảo trì và dễ mở rộng hơn. ```
